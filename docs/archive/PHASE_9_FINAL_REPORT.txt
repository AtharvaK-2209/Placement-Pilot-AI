2. PROBLEM STATEMENT
================================================================================

CHALLENGE:
Users who fall behind schedule have no automated recovery mechanism. They
continue following the original roadmap even when it's clear they won't finish
on time. This leads to:
- Missed deadlines
- Decreased motivation
- Abandoned goals
- No clear path to recovery

REQUIREMENT:
Implement an intelligent system that:
1. Detects when users are unlikely to meet their deadline
2. Generates an aggressive but achievable recovery plan
3. Provides specific, actionable steps
4. Preserves the original roadmap for safety
5. Works seamlessly with existing systems

15. KEY DESIGN DECISIONS & RATIONALE
================================================================================

DECISION 1: Deterministic Activation (No AI)
RATIONALE:
- Fast: < 5ms vs ~500ms AI call
- Free: Zero token cost
- Predictable: Consistent behavior
- Debuggable: Easy to trace logic

DECISION 2: Single AI Request (Not Multiple)
RATIONALE:
- Faster: One round-trip vs multiple
- Cheaper: ~1000 tokens vs 3000+
- Atomic: All-or-nothing generation
- Simpler cache: Single key

DECISION 3: Separate Storage (Not Modify Roadmap)
RATIONALE:
- Safe: Original roadmap protected
- Reversible: Easy to deactivate
- Clear: Data model separation
- Debuggable: No corruption risk

DECISION 4: Supplement (Not Regenerate)
RATIONALE:
- Faster: No full roadmap generation
- Preserves work: Completed weeks intact
- Less disruptive: Familiar structure
- Focused: Only changes what's needed
