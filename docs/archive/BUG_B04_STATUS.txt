================================================================================
                           THE PROBLEM
================================================================================

Error Message:
  Function setDoc() called with invalid data.
  Unsupported field value: undefined
  (found in field days.`w1-d1`.completedAt)

Root Cause:
  Application was writing objects with undefined values to Firestore.
  Firestore strictly validates and rejects undefined fields.

Why It Appeared:
  - Bug B-03 fixed Firestore permissions
  - Writes could now reach Firestore validation
  - Validation caught undefined values
  - Bug B-04 was exposed

When It Occurred:
  User clicks task checkbox → completeTask() called → updateTask() sends
  undefined values to Firestore → Firestore rejects → Error thrown

Impact:
  - Task completion could not persist
  - XP system broken
  - Progress tracking broken
  - Week unlocking broken
  - All downstream features blocked

================================================================================
                           THE SOLUTION
================================================================================

1. Added Sanitization Function
   File: src/repositories/FirestoreProgressRepository.ts
   Purpose: Remove all undefined values before Firestore write
   Method: Recursive traversal of object tree
   
2. Fixed Task Completion Logic
   File: src/repositories/FirestoreProgressRepository.ts (Line 169)
   Before: completedAt: completed ? "..." : undefined
   After:  Only include completedAt when completed=true
   
3. Fixed Day Completion Logic
   File: src/repositories/FirestoreProgressRepository.ts (Line 183)
   Before: completedAt: allDone ? "..." : day.completedAt
   After:  Use spread operator to conditionally include property
   
4. Applied Same Fixes to LocalStorage
   File: src/repositories/LocalStorageProgressRepository.ts
   Purpose: Maintain consistency across storage implementations
