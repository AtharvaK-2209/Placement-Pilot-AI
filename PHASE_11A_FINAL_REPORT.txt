================================================================================
                          ARCHITECTURE HIGHLIGHTS
================================================================================

1. Multi-Layer Caching System
   -------------------------------
   Memory Cache     → <1ms hit time
   Firestore Cache  → ~500ms hit time
   LocalStorage     → ~50ms hit time
   Gemini API       → 2-5s (only on cache miss)

   Result: ~90% reduction in API costs after warm-up

2. Auth-Aware Repository Pattern
   -------------------------------
   Authenticated users  → FirestoreXXXRepository (cloud sync)
   Guest users         → LocalStorageXXXRepository (offline-first)

   Result: Seamless data migration on sign-in

3. Immutable Versioning System
   -------------------------------
   Roadmap versions stored as v1, v2, v3...
   Active version tracked via pointer document
   Rollback support built-in

   Result: Never lose data, full audit trail

4. State Persistence Strategy
   -------------------------------
   Router State      → Primary (fast page transitions)
   SessionStorage   → Recovery (page refresh)
   Firestore        → Long-term (cloud sync)
   LocalStorage     → Fallback (offline support)

   Result: Zero data loss across sessions

5. Request Deduplication
   -------------------------------
   Multiple simultaneous requests → Single Gemini call
   In-flight tracker prevents duplicate API calls

   Result: No wasted API quota

6. Exponential Backoff Retry
   -------------------------------
   Transient errors (500, 503, network) → Retry with backoff
   Non-retryable errors (429, validation) → Fail fast

   Result: Resilient to temporary outages
